Classification Breakdown

How it works: Phase 1 scans recent memories for semantic similarity candidates (cosine threshold). Phase 2 sends candidates to an LLM for classification: DUPLICATE (merge), OVERLAPPING (partial), SUPPORTING (keep both), or CONFLICTING (flag). Phase 3 auto-resolves above-confidence candidates; rest go to review queue. Legacy backlog: 1,841 pre-pipeline candidates draining through the new system.
